# Project: Real-Time Crypto Anomaly Detection

I want to build a streaming pipeline that:
- Monitors live Bitcoin/Ethereum trades from Binance WebSocket
- Detects market manipulation (pumps, dumps, unusual patterns) with ML
- Visualizes in real-time Grafana dashboards
- Uses Kafka for streaming
- Deploys database to AWS RDS (free tier) (optional only if can and we have time)

**The Challenge:**
Binance sends 500-1000 trades/sec = 7-10 GB/day. AWS free tier is only 20 GB. I need to store the important stuff (anomalies) while controlling costs.

**My Questions:**
1. How should I architect this to stay free but still be production-quality?
2. What's the best data sampling/retention strategy?
3. What ML approach works best for real-time crypto anomalies?

**My Constraints:**
- Must be 100% free (AWS free tier only)
- Must use Kafka and Grafana specifically
- Must actually stream live data (not replay CSV)

Help me design this system, then we'll build it together step by step.
